对中西文化的一点思考

现在回头看川普的胜出，感觉完全是历史的必然。美国虽然强大，但做不到什么都管，世界警察不是那么容易当的。随着历史的发展，中国的崛起和美国的紧缩早已发生，只是很多人，包括希拉里，都不愿意承认这个事实。甚至很多中国人也没有想到这一点。起码很多中国人还是会觉得美国好，美国强大。

看看中国和美国的历史文化，美国实在太low了，根本没有个p。美国、加拿大别说跟中国，就是跟欧洲的多数国家比起来都是完败，毕竟连国家都没有几百年，无论如何也没法跟几千年的历史文化大国相比。可是，西方国家胜也是胜在这一点。几千年君君臣臣的官本位固化了人民的思想，约束了创造力。因为老祖宗总是对的，领导说的就是对的，一定要按照考纲来，所有的中国人都要学会带着枷锁跳舞。所以“上有政策，下有对策”是普遍的中国人的处世哲学，因为中国人不懂得反抗。西方国家的人，特别像法国这样的，没啥事也要示威游行一下，表达一下自己的意愿。中国人的意愿，一般只是鼓掌赞成。没有人会想着，政策制定的对不对。这样的思维和处世方式大大限制了国人的创造力。一旦有“出轨”的想法，第一时间会被打压。所以，更多人不愿意有自己的思维，自己的判断。哪怕有，也是“在心里”。这样的文化的另一个后果是缺少自己的判断，容易被煽动。因为没有自己的想法，只要有一个看似“权威”的人向某个方向引导，又给出了合理的解释，煽动就变得容易。这样的人群容易被带跑偏，就容易被人利用，哪怕初心是好的，也变成了别人的工具。这样的例子屡见不鲜。

出于这方面的考虑，党不得不做一些措施，封锁一些负面的、不良的、煽动的信息。这些措施的本意良善，可执行时也难免为奸人利用，做出对不起人民的事，抹黑了原本的善意。例如净网，本意只是避免黄色信息影响青少年，封锁反动信息，只需要针对性的屏蔽即可。执行者的无良却做成了闭网锁国，黄色信息任然满天飞。同时隔绝了先进科学技术信息的进入，这样的环境恶化了创新思维的土壤。所以中国在科研方面给人的感觉还是学习引进非常厉害，创新创造随人之后。

给博客添加评论功能

给博客添加评论功能

建了一网站，你会希望有人评论。Octopress自带disqus的整合功能。下面说说怎么添加这个评论功能。

准备工作

  • 注册Disqus
    • 你也可以不注册，直接用google或者facebook账号登陆就行。
  • 然后选择 install disqus(或者去https://disqus.com/profile/signup/intent/)，选择I want to install Disqus on my site。
  • 然后一推配置，随便你选就行了。

安装Disqus

也就是把_config.yml文件中的disqus_show_comment_count:改成true,在disqus_short_name:加上disqus名字就行了。

用Octopress在GitHub发布博客

Octopress可以配合github或者heroku使用。Github还是最方便的，用户可以有无限的public空间。

配置Octopress前准备github

  • 注册github账号
  • 建一个新的repository。
    • 如果是github主站，域名是username.github.io, 那么repository的名字就是username.github.io。
    • 如果不是主站，而是项目页（gh-pages），域名是https://username.github.io／repository_name, 那么repository的名字就是repository_name。

配置Octopress

这个命令会:

  • 问你github的url。
    • 如果是github主站，域名是username.github.io, 那么输入git@github.com:username/username.github.io.git。
    • 如果不是主站，而是项目页（gh-pages），域名是https://username.github.io／repository_name, 那么输入http://username.github.io/repository_name。
  • Rename the remote pointing to imathis/octopress from ‘origin’ to ‘octopress’
  • Add your Github Pages repository as the default origin remote.
  • Switch the active branch from master to source.
  • Configure your blog’s url according to your repository.
  • Setup a master branch in the _deploy directory for deployment.

写完以后，就可以发布博文了。

这个使你可以在本地预览博客页面。运行后，在浏览器打开http://localhost:4000/就可以看到你的页面了。
